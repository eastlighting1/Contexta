# 보안 정책

## 지원 버전

보안 패치는 최신 릴리스에만 적용됩니다.
이전 버전은 별도로 패치하지 않습니다.

| 버전 | 지원 여부 |
|------|----------|
| 최신 버전 | 지원 |
| 이전 버전 | 미지원 |

## 취약점 신고 방법

**보안 취약점은 공개 GitHub 이슈로 등록하지 마세요.**

취약점은 메인테이너에게 이메일로 직접 제보해 주세요.
제보 시 아래 내용을 최대한 포함해 주시면 빠른 대응에 도움이 됩니다.

- 취약점 설명 및 잠재적 영향 범위
- 재현 방법 또는 최소 재현 코드(PoC)
- 파악하고 계신 수정 방안이나 완화 방법(있는 경우)

### 대응 일정

| 단계 | 목표 기한 |
|------|----------|
| 접수 확인 | 영업일 기준 3일 이내 |
| 초기 평가 | 영업일 기준 7일 이내 |
| 수정 또는 완화 조치 | 심각도에 따라 상이 (아래 참고) |

### 심각도별 수정 목표

| 심각도 | 수정 목표 기한 |
|--------|--------------|
| 치명적(Critical) | 7일 이내 |
| 높음(High) | 14일 이내 |
| 보통(Medium) | 다음 정기 릴리스 |
| 낮음(Low) | 메인테이너 재량에 따라 예정 |

## 적용 범위

Contexta는 **로컬 우선(local-first)** 관측 라이브러리입니다.
모든 데이터는 사용자 자신의 기기에만 저장되며, 클라우드 백엔드·텔레메트리·기본 네트워크 트래픽이 없습니다.

따라서 아래 항목은 **적용 범위 외**입니다.

- 사용자 기기에 물리적 접근이 필요한 공격
- 선택적 서드파티 통합(scikit-learn, PyTorch, Transformers, MLflow, OpenTelemetry)의 취약점 — 해당 프로젝트에 직접 신고해 주세요
- 핵심 라이브러리에 영향을 주지 않는 `examples/` 예제 스크립트 문제

## 공개 정책

패치가 릴리스되면 GitHub 저장소에 간략한 보안 권고(Security Advisory)를 게시합니다.
저희는 **협력적 공개(coordinated disclosure)** 원칙을 따르며, 신고자의 동의를 받아 권고문에 공헌자로 기재합니다.
