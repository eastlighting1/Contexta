# Contexta 주요 기능

`Contexta`는 로컬 우선(local-first) ML 관측성(observability) 워크플로를 위한 단일 제품 인터페이스입니다.

`Contexta`는 캡처, 저장, 쿼리, 비교, 리포팅 및 복구를 별개의 도구로 취급하는 대신, 하나의 표준화된 제품 정체성 아래로 통합합니다:

- Python 임포트 루트: `contexta`
- CLI 대상: `contexta`
- 환경 변수 접두사: `CONTEXTA_*`
- 워크스페이스 루트: `.contexta/`

## Contexta란 무엇인가요?

`Contexta`는 다음과 같은 팀을 위해 설계되었습니다:

- 관측성 데이터를 로컬에 유지하고 조사 가능하게 만들고 싶을 때
- 임시 페이로드 대신 표준화된 모델(canonical models)을 사용하고 싶을 때
- 데이터를 쓴 바로 그 제품 인터페이스에서 실행 결과를 쿼리하고 비교하고 싶을 때
- 복구 기능을 외부 스크립트가 아닌 제품 내부의 기능으로 유지하고 싶을 때

지향하는 핵심 루프는 다음과 같습니다:

1. 캡처 (capture)
2. 저장 (store)
3. 쿼리 (query)
4. 리포트 (report)

제품의 다른 모든 기능은 이 루프를 신뢰할 수 있고 조사 가능하게 만드는 것을 중심으로 조직되어 있습니다.

## 주요 제품 성과

### 6개의 별도 인터페이스 대신 하나의 제품으로 통합

`Contexta`는 여러 독립적인 주요 인터페이스들을 하나의 표준화된 인터페이스로 대체합니다.

이로 인해 얻을 수 있는 이점:

- 하나의 공용 제품 정체성
- 하나의 공용 임포트 루트
- 하나의 공용 워크스페이스 구조
- 새로운 코드를 작성할 때 가장 먼저 확인해야 할 한 곳

### 기본이 되는 로컬 우선(Local-First) 방식

제품은 로컬 워크스페이스와 로컬 조사를 중심으로 구축되었습니다.

이는 다음을 의미합니다:

- 원격 서비스 없이도 데이터를 생성하고 조사할 수 있습니다.
- 표준 데이터는 사용자가 제어하는 워크스페이스에 존재합니다.
- 쿼리와 리포팅을 이해하기 위해 호스팅된 플랫폼이 필요하지 않습니다.

내장된 HTTP/UI가 전달 인터페이스로 존재할 수 있지만, 이들 또한 설계상 로컬 우선 및 로컬 전용 방식입니다.

### 임시 로그 대신 표준화된 데이터(Canonical Data) 사용

`Contexta`는 다음 항목들에 대한 표준화된 모델을 중심으로 조직되어 있습니다:

- 맥락 (context)
- 레코드 (records)
- 아티팩트 (artifacts)
- 리니지 및 프로버넌스 (lineage and provenance)

이것이 중요한 이유:

- 쿼리 결과가 더 안정적입니다.
- 예측 가능한 스키마로부터 리포트를 빌드할 수 있습니다.
- 성능 저하 또는 부분적인 상태(Degraded)를 자유 형식의 텍스트 뒤에 숨기지 않고 명시적으로 표현할 수 있습니다.

### 통합된 읽기 및 조사 흐름

워크스페이스에 표준 데이터가 존재하면, `Contexta`는 하나의 조사 인터페이스를 통해 이를 읽을 수 있습니다.

여기에는 다음 내용이 포함됩니다:

- 실행 목록 조회
- 실행 스냅샷
- 비교
- 진단
- 리니지 순회
- 리포트 생성

단순히 데이터를 저장하는 것뿐만 아니라, 일관된 방식으로 조사할 수 있게 만드는 것이 핵심입니다.

### 복구 또한 제품의 일부입니다

`Contexta`는 복구를 사후 고려 사항이 아닌 일급 관심사(first-class concern)로 취급합니다.

여기에는 다음 내용이 포함됩니다:

- 재생 (replay)
- 백업 (backup)
- 복구 (restore)

관측성 시스템은 복구가 커스텀 스크립트와 문서화되지 않은 운영자 지식에 의존할 때 신뢰하기 어려워지기 때문에 이 기능은 중요합니다.

## 기능 맵 (Feature Map)

| 기능 영역 | 가치 |
| --- | --- |
| 통합된 퍼사드 | 제품을 위한 단일 시작점 제공 |
| 로컬 워크스페이스 | 조사 가능하고 로컬 우선인 표준 저장소 |
| 표준 계약 | 안정적인 모델, 검증 및 직렬화 |
| 조사 계층 | 쿼리, 비교, 진단, 리니지, 리포트 |
| 복구 계층 | 재생, 백업, 복구 |

## 누구에게 가장 도움이 되나요?

### 신규 사용자

신규 사용자는 다음 이점을 누릴 수 있습니다:

- 시작할 수 있는 한 곳
- 배워야 할 하나의 명명 시스템
- 퀵스타트에서 심화 사용으로 이어지는 명확한 경로

### 운영자

운영자는 다음 이점을 누릴 수 있습니다:

- 명확한 워크스페이스 소유권
- 동일한 제품 스토리 내의 복구 기능
- 공용 인터페이스와 내부 구현 간의 더 명확한 경계

## 이 섹션에서 약속하지 않는 것

이 섹션은 제품의 방향과 안정적인 공용 구조를 설명합니다. 모든 릴리스 정렬 세부 사항이 이미 완료되었다고 주장하는 것은 아닙니다.

현재 프로토타입 단계에서:

- 표준 명칭은 이미 `Contexta` / `contexta`로 적용되었습니다.
- 일부 패키징 및 CLI 정렬 작업이 여전히 진행 중입니다.
- 공용 문서 세트가 계속해서 채워지고 있습니다.
